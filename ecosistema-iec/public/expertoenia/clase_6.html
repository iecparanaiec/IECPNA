DE PROMPT CASUAL A PROMPT PROFESIONAL

Prompt casual

“Haceme un flyer.”

Depende demasiado de lo que la IA interprete.

Prompt profesional

“Creá un flyer vertical para promocionar un curso de IA dirigido a adultos. Usá tono cercano, colores azul y violeta, título breve, tres beneficios y un llamado a la acción.”

Reduce la ambigüedad y mejora el control.

¿QUÉ ES JSON?

JSON es una forma ordenada de representar información mediante pares de nombre y valor.

🏷️

Clave

El nombre del dato.

📦

Valor

La información asociada.

{ }

Objeto

El conjunto organizado.

JSON EXPLICADO SIN PROGRAMAR

{ "nombre": "Curso de IA", "publico": "Adultos y jóvenes", "objetivo": "Promocionar la inscripción", "formato": "Flyer vertical" }

Clave

“nombre”, “público”, “objetivo”, “formato”.

Valor

La información concreta de cada campo.

REGLAS BÁSICAS DE JSON

Se usa

• Llaves para abrir y cerrar.

• Claves entre comillas.

• Dos puntos entre clave y valor.

• Comas entre elementos.

• Corchetes para listas.

Se evita

• Coma final innecesaria.

• Comillas faltantes.

• Llaves sin cerrar.

• Mezclar instrucciones fuera del objeto.

• Claves ambiguas.

PROMPT PARA PEDIR JSON

“Convertí el siguiente pedido en un JSON claro, válido y profesional para utilizarlo como prompt en otra herramienta de IA. Incluí contexto, objetivo, público, tarea, formato, tono, restricciones, elementos obligatorios y criterios de calidad. No agregues explicaciones fuera del JSON.”

ESTRUCTURA JSON PARA PROMPTS

{ "contexto": "Descripción de la situación", "rol": "Especialista que debe representar la IA", "objetivo": "Resultado principal esperado", "publico_objetivo": "Personas destinatarias", "tarea": "Acción concreta", "formato_salida": "Tipo de resultado", "tono": "Estilo de comunicación", "restricciones": [ "Límite o condición 1", "Límite o condición 2" ], "criterios_calidad": [ "Criterio 1", "Criterio 2" ] }

EJEMPLO JSON PARA CANVA

{ "herramienta": "Canva IA", "objetivo": "Crear un flyer promocional", "publico_objetivo": "Adultos interesados en IA", "formato": "Flyer vertical para Instagram", "estilo_visual": "Moderno y profesional", "colores": ["#38bdf8", "#818cf8", "#0a0e1a"], "contenido_obligatorio": [ "Título", "Tres beneficios", "Llamado a la acción" ] }

EJEMPLO JSON PARA LOVABLE

{ "herramienta": "Lovable", "tipo_proyecto": "Página web institucional", "objetivo": "Generar consultas comerciales", "secciones": [ "Inicio", "Servicios", "Beneficios", "Preguntas frecuentes", "Contacto" ], "funcionalidades": [ "Formulario", "Botón de WhatsApp", "Diseño responsivo" ] }

JSON NO GARANTIZA UN BUEN RESULTADO

Un JSON bien escrito puede contener una mala idea, información incompleta o instrucciones contradictorias.

🔎

Revisar contenido

No sólo la sintaxis.

🧪

Probar resultado

Comparar versiones.

🔄

Mejorar estructura

Corregir campos y criterios.

CHECKLIST DE UN JSON CONFIABLE

✔ Tiene un objetivo claro.

✔ Define la herramienta destino.

✔ Incluye contexto suficiente.

✔ Evita contradicciones.

✔ Define formato y tono.

✔ Especifica restricciones.

✔ Incluye criterios de calidad.

✔ Es JSON válido.

PRIVACIDAD AL COMPARTIR PANTALLA

✅

Antes de compartir

• Cerrar correos y chats privados.

• Ocultar contraseñas.

• Elegir una sola ventana.

• Revisar notificaciones.

• Evitar datos de clientes.

🚫

Nunca mostrar

• Datos bancarios.

• Claves o códigos.

• Historias clínicas.

• Documentos confidenciales.

• Información personal ajena.
